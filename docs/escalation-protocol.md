# Human-in-the-Loop Escalation Protocol — Mistral 7B RAG + Agent Pipeline

**Component:** C3 Challenge Extension
**Purpose:** Define when and how a human must review or override system outputs
before they are returned to users or acted upon.

---

## Overview

Not all RAG outputs are equal risk. This protocol classifies outputs by risk level
and defines escalation paths for high-risk cases where automated guardrails are
insufficient. The protocol applies to both RAG mode and Agent mode.

---

## Risk Classification

| Risk Level | Trigger Condition | Action |
|---|---|---|
| LOW | top-1 similarity >= 0.80, no anomaly flag | Return answer automatically |
| MEDIUM | similarity 0.70-0.80, or query > 100 tokens | Add disclaimer, log for review |
| HIGH | similarity < 0.70, or anomaly flag = 1 | Queue for human review before returning |
| CRITICAL | similarity < 0.50, or toxicity flag | Block response, notify on-call immediately |

---

## Escalation Paths

### Path 1: Automated (LOW risk)
1. Query arrives
2. Retrieval similarity >= 0.80
3. Answer generated by Mistral 7B
4. Response returned to user with source citations
5. Logged to audit trail

### Path 2: Flagged with Disclaimer (MEDIUM risk)
1. Query arrives
2. Retrieval similarity 0.70-0.80
3. Answer generated by Mistral 7B
4. Disclaimer appended: "This answer is based on limited source matches.
   Verify with primary sources before acting on this information."
5. Response returned to user
6. Logged to review queue for daily human spot-check

### Path 3: Human Review Queue (HIGH risk)
1. Query arrives
2. Retrieval similarity < 0.70 OR anomaly flag = 1
3. Answer generated but NOT returned immediately
4. Answer placed in human review queue (Slack notification to on-call)
5. Human reviewer has 30 minutes to approve, edit, or reject
6. If approved: return to user with HIGH_RISK label
7. If rejected: return standard fallback message
8. All decisions logged with reviewer ID and timestamp

### Path 4: Immediate Block (CRITICAL risk)
1. Query arrives
2. Retrieval similarity < 0.50 OR output contains toxicity markers
3. Response BLOCKED — not generated or discarded
4. User receives: "I cannot answer this query reliably. Please consult
   a subject matter expert."
5. PagerDuty alert fires immediately to on-call engineer
6. Incident logged with full query context for post-mortem

## Implementation Requirements

| Component | Requirement |
|---|---|
| Retrieval threshold check | Add confidence check after ChromaDB query in agent_controller.py |
| Review queue | Slack webhook or simple database table with status field |
| Reviewer UI | Minimal web form: show query, retrieved docs, proposed answer, approve/reject |
| Audit logging | Extend logs/audit-trail.json with per-response risk classification |
| SLA | HIGH risk reviews must complete within 30 minutes during business hours |
| Fallback | If reviewer unavailable after 30 min, return MEDIUM-risk response with disclaimer |

## Integration with Monitoring

The Prometheus dashboard (C1) already tracks:
- agent_retrieval_top_similarity: feeds directly into risk classification
- agent_input_anomaly_flag: triggers HIGH or CRITICAL path
- agent_requests_total with status labels: tracks escalation rates

New metrics to add for full HITL visibility:
- hitl_queue_depth: number of responses awaiting human review
- hitl_review_latency_seconds: time from queue entry to decision
- hitl_rejection_rate: fraction of HIGH-risk responses rejected by reviewers

## Connection to Risk Register

This protocol directly mitigates:
- R03 (corpus hallucination): HIGH/CRITICAL paths intercept OOD responses
- R09 (no AI disclaimer): MEDIUM path enforces disclaimer
- R06 (prompt injection): CRITICAL path blocks suspicious outputs
- R05 (compliance): all escalations logged with full audit trail
